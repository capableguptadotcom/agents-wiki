Thesis

Agents become product actors only when identity, tools, memory, approvals, workflows, and evals are first-class.

Architecture map

Agent-native products are execution systems, not chat boxes.

The user may start with voice, chat, Slack, or a button inside a workflow. The product must still convert that request into governed actions, durable execution, observable state, and recoverable outcomes.

Runtime split Agent reasons. Workflow recovers. Tool gateway enforces.
Source of truth Product DB and domain systems stay authoritative.
Trust model Approvals, audits, scopes, and evals build confidence.

Release bundle

A release should pin prompts, model IDs, tool versions, memory schemas, policies, eval sets, and workflow code. Without this, you cannot reproduce behavior after a failure.
